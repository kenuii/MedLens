# MedLens Project Logbook  

---

## 🗓️ 2025-11: November  

### **Week 1: Project Idea & Research**  
- **Focus:** Identifying the Problem  
- **Details:** I began developing MedLens after recognizing how critical early detection is for conditions like skin cancer and tumors. I researched how delayed diagnosis impacts survival rates and explored how AI is currently used in medical imaging. I reviewed trusted medical sources and studied how different conditions appear visually.  
- **Challenges:** I initially had multiple project ideas, but I narrowed my focus to image-based detection to create a clear and impactful solution.  

---

### **Week 2: Planning & System Design**  
- **Focus:** Feature Planning and User Flow  
- **Details:** I defined the core features: image upload, AI analysis, risk classification, explanations, and recommended actions. I mapped out the full user journey from capturing an image to receiving results.  
- **Challenges:** Balancing simplicity with functionality was difficult. I focused on minimizing steps so the app could be used quickly and efficiently.  

---

### **Week 3: UI Development (FlutterFlow)**  
- **Focus:** Frontend Implementation  
- **Details:** I began building the app interface using FlutterFlow. I created the upload page, preview screen, and results page, ensuring smooth navigation between them.  
- **Challenges:** I faced issues with image preview and layout responsiveness across devices, which I resolved by adjusting widget settings and layout structures.  

---

## 🗓️ 2025-12: December  

### **Week 4: Firebase Integration**  
- **Focus:** Backend Setup  
- **Details:** I integrated Firebase for cloud storage and backend functionality. I implemented image upload and worked on retrieving image URLs for processing.  
- **Challenges:** Firebase storage permission errors caused upload failures. I fixed this by adjusting security rules and verifying file paths.  

---

### **Week 5: AI Model Training**  
- **Focus:** Building the Model  
- **Details:** I trained my own image classification model using over 10,000 medical images, including various skin conditions and tumors. I focused on proper labeling and dataset organization.  
- **Challenges:** Managing and balancing the dataset was challenging, as uneven data could reduce accuracy.  

---

### **Week 6: AI Integration Planning**  
- **Focus:** System Architecture  
- **Details:** I designed how the AI would connect to the app, including how results would be structured (condition, risk level, explanation, action).  
- **Challenges:** Understanding how to pass image data into the AI system required additional research and testing.  

---

## 🗓️ 2026-01: January  

### **Week 7: AI Integration**  
- **Focus:** Connecting AI to App  
- **Details:** I connected the AI model to the app so that uploaded images could be analyzed and results returned.  
- **Challenges:** I encountered issues with image formats and data flow, which I resolved through debugging and testing.  

---

### **Week 8: Results System Development**  
- **Focus:** Output & User Feedback  
- **Details:** I implemented risk levels, explanations, and recommended actions. I ensured the results were easy to understand for all users.  
- **Challenges:** Simplifying complex medical information while keeping it accurate required multiple revisions.  

---
